Sales Return Entry — Step-by-Step Workflow

Credit-note return flow from the dashboard customer quick action through local persistence and Zoho sync.

flowchart TD
    A([User taps a customer on Dashboard]) --> B[Client Operations Sheet opens]
    B --> C[Tap Sales Return Credit Note action]
    C --> D[Open Sales Return Dialog]

    D --> E["Load customer local invoices
collect purchased item IDs
filter catalog to purchased items"]
    E --> F{Customer has
purchase history?}
    F -- No --> G["Show warning:
returns allowed only for
previously sold items"]
    G --> H([CONFIRM disabled - exit])

    F -- Yes --> I[Select returned item
from dropdown]
    I --> J["Find invoices containing that item
sort newest first
create a qty field per invoice"]
    J --> K[Enter return quantity per invoice]
    K --> L{"Each qty valid?
0 to sold units"}
    L -- No --> M["Inline field error
such as Max N"]
    M --> K

    L -- Yes --> N[Tap CONFIRM RETURN]
    N --> O{"Form valid AND
total qty above zero?"}
    O -- No --> P[Show error snackbar]
    P --> K

    O -- Yes --> Q["Build SalesReturn payload
temp id plus temp credit-note number
returned line items plus reason"]
    Q --> R[("Save return to local Hive
restore van stock instantly")]
    R --> S[("Enqueue SyncQueueItem
type return, status pending")]
    S --> T[Trigger SyncWorker syncPendingItems]
    T --> U["Close dialog
success snackbar
onReturnConfirmed callback"]

    U --> V{Network available?}
    V -- No --> W[Item stays queued
retried on reconnect]
    V -- Yes --> X[POST credit note to
Zoho Books via syncSalesReturn]
    X --> Y{Zoho accepted?}
    Y -- Yes --> Z(["Mark synced
replace temp id with Zoho id"])
    Y -- No --> AA[Mark failed
stays in queue for retry]
